# 从这里开始看

如果你没有开发基础，先不要看代码，按下面顺序看：

## 1. 先看这个文件

你现在打开的就是最适合非技术用户阅读的说明。

## 2. 再打开本地演示

双击项目根目录里的 `OPEN_DEMO.bat`。

它会用系统默认浏览器打开一个本地演示。

这一步不要求安装 Chrome。

如果你想看“为什么现在还不能直接出 APK”，也可以双击：

`CHECK_ANDROID_ENV.bat`

它会直接告诉你这台机器是否缺 Java、Gradle Wrapper 或 Android SDK。

如果你想走“不在本机装 Android 开发环境”的路线，再看：

`docs/process/remote-build.md`

## 3. 演示里你能看到什么

- 六艺选择
- 式的选择
- 手动开始一组
- 手动结束一组
- 按 `2-1-2` 节奏实时显示当前阶段与秒数
- 本次训练里累计了多少组、多少次

## 4. 这和最终 Android App 的关系

这个演示不是最终 Android 成品，只是为了让你先看懂：

- 需求有没有被理解对
- 训练逻辑是不是符合你的意思
- “开始/结束一组后自动算次数”的规则是不是你要的

如果演示逻辑没问题，我下一轮就把同样的逻辑继续往 Android 正式实现里推进。

## 4.1 以后怎么测试真正成品

你以后不需要学 Android 开发环境。

更合理的方式是：

- 我产出 APK
- 你在安卓手机上安装
- 你直接试用并反馈

在那之前，本地演示只负责把需求和逻辑先确认清楚。

## 5. 这轮你最应该关注的三件事

### A. 节奏与计次逻辑

当前定义是：

- 你按“开始一组”时开始计时
- 你按“结束一组”时停止计时
- 系统按固定 `2-1-2` 节奏实时推进阶段
- 每完成一个完整节奏循环，当前组次数加一

### B. 训练记录逻辑

当前定义是：

- 一批训练可以有多组
- 每组都有自己的开始时间、结束时间、持续时间、次数
- 本次训练结束后会有总组数、总次数以及日期时间

### C. 动作标准内容

这一部分目前只搭好了结构，还没有正式导入书籍对应的标准文字和示意图。

原因不是技术问题，而是内容来源和版权边界需要谨慎处理。

## 6. 你现在怎么给我反馈最有效

你只需要直接用自然语言告诉我这几件事：

1. 这个“开始一组/结束一组/按 2-1-2 实时节奏引导和计次”的逻辑是不是你要的
2. 你希望一批训练是“单动作多组”还是“多动作混合”优先
3. 动作标准页里你最想先看到哪些内容

你不需要懂代码，我来负责把你的意思继续落进工程里。
