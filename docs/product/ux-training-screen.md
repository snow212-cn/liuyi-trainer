# 训练执行流设计原则 v0.2

## 1. 页面目标

训练流不是“首页直接跳进去自动开始”，而是：

1. 首页统一完成六艺十式选择
2. 进入训练准备页
3. 用户手动点“开始本组”
4. 再进入训练执行页
5. 结束本组后进入休息页
6. 结束本次训练后进入可编辑总结页
7. 用户确认修正后再保存进历史

训练执行页本身不是信息总览页，而是节奏引导页。

它最重要的工作只有三个：

- 告诉用户当前该做什么动作阶段
- 告诉用户当前节奏秒数
- 告诉用户当前组已经做了多少次

## 2. 信息层级

### 一级信息

必须最大、最醒目：

- 当前阶段：下落 / 停 / 起
- 当前阶段秒数
- 当前组次数

建议表现：

- 占据屏幕主体区域
- 粗字重
- 强对比度颜色
- 必要时配合轻微呼吸式动画或脉冲效果

### 二级信息

需要能看到，但不应抢主视觉：

- 第几组
- 本次训练已完成多少组
- 当前动作名称
- 当前式名称

建议表现：

- 放在顶部或底部条带
- 中等字号
- 低于一级信息的对比度

### 三级信息

仅供补充，不应打断训练：

- 本次训练起始时间
- 累计训练时长
- 帮助提示

建议表现：

- 小字号
- 弱化颜色
- 默认不占据中央区域

## 3. 视觉可配置项

为了适应不同用户，训练页应允许切换视觉风格：

- 标准模式：平衡信息和装饰
- 极简模式：只保留一级和少量二级信息
- 高对比模式：适合远距离、快速扫视
- 强提示模式：阶段切换时强化颜色、动画和声音

首版不需要做复杂皮肤系统，但应该预留设置入口。

## 4. 训练准备页

训练准备页必须存在。

它至少要提供：

- 当前动作与当前式
- 当前节奏
- 当前休息预设
- 语音提示开关
- “开始本组”按钮

不能在用户刚进入训练页时就自动开练。

## 5. 组间休息设计

组间休息是训练执行流程的一部分，不应交给用户自行心算。

建议默认流程：

1. 结束本组
2. 自动进入休息页或休息面板
3. 显示建议休息倒计时
4. 倒计时结束后，如果用户还未开始下一组，则自动转为正计时

## 6. 为什么倒计时转正计时是合理的

合理，而且很实用。

- 倒计时能告诉用户“建议还剩多久”
- 转为正计时能告诉用户“已经多休息了多久”
- 对训练记录和自我约束都更直观

所以不建议二选一，而建议两者连续衔接。

## 7. 休息时长怎么做更合理

首版最合理的方案不是复杂算法，而是“用户可设置 + 提供推荐值”。

推荐实现：

- 提供几个常用预设：30 秒、60 秒、90 秒、120 秒、180 秒
- 允许用户改成自定义秒数
- 允许为某一动作保存默认休息时长

## 8. 训练总结页

训练总结页不是只读页。

首版必须支持：

- 显示每组次数
- 保存前逐组修正次数
- 自动重新汇总总次数
- 用户手动点保存后再进入历史记录

## 9. 训练历史页

训练历史列表页必须轻量。

列表页每条只显示：

- 动作名与式名
- 日期时间
- 总组数和总次数
- 每组次数摘要

详细内容放到点击后的详情页，不允许一条记录直接霸占整屏列表。

## 10. 首版默认建议

如果暂时没有用户自己的偏好，建议默认：

- 难度较低、次数较多的组：60 秒
- 中等强度组：90 秒
- 高强度或接近极限组：120 到 180 秒

这不是硬规则，只是默认建议。

## 11. 页面结构建议

训练执行页建议拆为两个主状态：

- 动作进行中
- 组间休息中

不要把两者混在一套视觉层级里，否则界面会变乱。
