# 测试与验证策略 v0.1

## 1. 目标

在用户没有 Android 开发环境、没有 Chrome、也不擅长技术操作的前提下，仍然让项目保持可验证、可推进。

## 2. 测试分层

### A. 规则验证

验证对象：

- 开始一组
- 结束一组
- `2-1-2` 实时节奏引导
- 实时计次
- 多组累计
- 本次训练汇总

手段：

- 轻量原型
- 领域模型函数
- 计算示例

### B. 人工体验验证

验证对象：

- 页面是否看得懂
- 训练流程是否符合预期
- 汇总信息是否足够

手段：

- 用户双击打开本地演示
- 用户直接反馈哪里不对

### C. Android 平台验证

验证对象：

- 真机安装
- 页面交互
- 后台恢复
- 本地存储

手段：

- 远程 CI 生成 APK
- 用户在 Android 手机上安装测试

## 3. 当前不采用的方案

- 不把 Chrome 作为必需安装项
- 不把 Playwright 自动化作为你验收项目的前提
- 不把 Android Studio 安装作为当前阶段的前置门槛

## 4. 当前推荐流程

1. 我先把功能逻辑做成轻量可看懂的原型。
2. 你确认逻辑是否符合你的真实训练习惯。
3. 逻辑稳定后，我再推进 Android 实现。
4. Android 版本需要安装测试时，优先给你 APK，而不是要求你搭开发环境。

## 5. 你未来如何测试成品

最合理的方式不是让你学会 Android 开发，而是：

1. 我产出 APK。
2. 你把 APK 安装到安卓手机上。
3. 你按真实训练流程试用。
4. 你只告诉我“哪里不顺、哪里不对、哪里缺东西”。

## 6. 当前阶段你只需要做什么

- 打开本地演示
- 看懂流程
- 确认功能逻辑

你现在不需要：

- 安装 Chrome
- 安装 Android Studio
- 学 Gradle
- 学 adb
