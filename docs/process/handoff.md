# 对话过长时的续做说明

日期：2026-03-27

## 这份文件是干什么的

这份文件专门用来解决“聊天上下文太长”的问题。

以后如果当前对话变得太长，直接新开一个对话，不要重讲全部历史。只要让新的 AI 先读取本文件和相关状态文档，就能继续。

## 新对话开头直接发这句

请先阅读 `docs/process/start-here.md`、`docs/process/current-status.md`、`docs/process/handoff.md`，然后按当前官方路线继续，不要偏离。

## 当前项目目标

做一款 Android 手机上的本地训练助手，服务“六艺十式”练习，核心不是自动识别动作，而是：

- 用户手动开始一组
- 用户手动结束一组
- 系统在训练中按 `2-1-2` 节奏实时显示阶段与秒数
- 每完成一个完整节奏循环，当前组次数加一
- 支持多组训练
- 组间休息先倒计时，超时后转正计时
- 训练结束后保存本次训练的日期时间、总组数、总次数、每组次数
- 动作标准页后续可接文字、示意图、注意事项

## 已经明确的产品规则

- “一批训练”统一改叫“本次训练”
- 实时训练页要突出：
  - 当前阶段
  - 当前阶段秒数
  - 当前组次数
- 低优先级信息弱化显示
- 休息时长允许用户选预设值
- 休息倒计时结束后自动切成正计时
- 动作标准页采用某一式的独立详情页
- 书中原文和原图不自动抓取；如需导入，走用户手动提供内容的路线

## 当前已落地成果

- 已有可本地打开的轻量演示：`OPEN_DEMO.bat`
- 演示已覆盖训练节奏、手动开始/结束一组、休息倒计时与本次训练汇总
- Android 工程已有首页、训练页、休息页、总结页、历史页、动作标准页
- Android `ViewModel` 已接管训练会话状态
- 已建立训练状态机与本地持久化入口
- 训练完成后会尝试写入历史记录
- 历史记录页已显示每次训练的分组明细
- 仓库已补齐 Gradle Wrapper 文件，但尚未完成首次 Android 构建
