# UI 重构蓝图 v2.1

日期：2026-03-29

## 1. 这轮重构的边界

本轮不是继续修小问题，而是重新校正界面气质与主页面结构。

当前只重点重构以下方向：

- 首页的整体风格与选择方式
- 训练进行页的实时信息呈现方式
- 历史列表页与历史详情页的可维护性预期
- 动作标准页的内容承载结构

当前明确暂不改动的地方：

- 组间休息页的功能结构
- 训练总结页的功能结构

说明：

- 这两页可以统一视觉语言，但不在本轮改页面逻辑。
- Android 正式页面必须以 `prototype/ui-redesign.html` 为 UI 基准，不允许脱离该稿自由发挥。

## 2. 新的视觉方向

目标不再是“干净的健康类 App”，而是更接近《囚徒健身》的训练气质。

关键词：

- 粗粝
- 工业感
- 牢房器械感
- 训练记录板
- 钢板、混凝土、锈色、粉笔记号

禁止继续出现的感觉：

- 轻飘飘的奶油卡片
- 像冥想或轻健身应用
- 大面积柔和留白
- 十个圆角按钮平铺的通用 UI

视觉语言原则：

- 大标题像器械铭牌或训练规章
- 一级信息要像钉在墙上的数字牌
- 次要信息缩小、变暗、退后
- 交互组件尽量像轨道、刻度尺、拨盘、钢牌，而不是普通按钮墙
- 动画应该像节奏器、脉冲灯、呼吸带，而不是花哨装饰

## 3. 首页

### 3.1 首页定位

首页应该是一块短而硬的“训练指挥台”，而不是长训练台。

用户进入首页后，应该在较短路径内完成：

- 选择六艺
- 选择十式
- 查看当前节奏
- 设定组间休息
- 进入训练
- 查看动作标准
- 查看训练历史

### 3.2 首页结构

首页改为四段：

1. 当前动作摘要钢牌
2. 六艺切换架
3. 十式紧凑切换区
4. 休息刻度与主操作区

### 3.3 六艺选择

六艺不再做成长卡片堆叠。

建议形式：

- 2 x 3 小型钢牌
- 或短标签架

要求：

- 一眼看到六艺全貌
- 当前选中项要明显加重
- 非选中项不抢视觉

### 3.4 十式选择

十式不再做成 10 个均质按钮矩阵。

改为：

- 当前式大号显示
- 用“上一式 / 当前式 / 下一式”三块相邻切换
- 同时显示当前进度，如 `4 / 10`

这样做的原因：

- 用户需要看到正式名称，但不需要一个很长的十项列表
- 当前式必须最突出，其他式只承担切换功能
- 十式名字来自原书，仍然必须进入首页选择层
- 不再出现十个数字按钮或十个均质按键同时竞争注意力

### 3.5 组间休息设置

休息设置不应该继续做一排大按钮。

改为更紧凑的形式：

- 刻度尺
- 拨盘
- 滑尺式预设条

首版可保留若干预设值：

- 45 秒
- 60 秒
- 90 秒
- 120 秒
- 自定

## 4. 训练准备页

训练准备页继续保留。

这页必须继续承担一个核心职责：

- 防止用户一进入训练页就自动开练

必须保留：

- 当前艺与式
- 节奏 `2-1-2`
- 语音提示状态
- 开始本组按钮
- 动作标准入口

## 5. 训练进行页

### 5.1 页面定位

训练进行页不是概览页，而是节奏引导页。

### 5.2 一级信息

必须永远最强：

- 当前阶段
- 当前阶段秒数
- 当前组次数

### 5.3 呈现方式

不能再只靠几段大字硬堆。

应加入图形化节奏引导：

- 环形节奏盘
- 呼吸脉冲带
- 发光节奏标记

建议组合：

1. 中央环形节奏盘显示当前节奏
2. 中央数字显示当前阶段秒数
3. 次要区域压缩成同一排，只显示：
   - 当前组次数
   - 累计次数
   - 已训练时长
4. 已完成组数退到更弱层级，不与主信息抢视觉

### 5.4 语音与提示

训练进行页要明确为后续语音提示预留位置。

首版文案以：

- `落`
- `停`
- `起`

为主。

同时应预留：

- 语音开关状态
- 呼吸声或节奏声提示位

### 5.5 信息层级

训练进行页只允许一个主视觉中心。

不允许再出现：

- 同等大小的信息块并排竞争
- 无法一眼看出“当前阶段秒数”的情况
- 为了说明设计而额外出现解释性文字块
- 低价值信息如开始时间占据主要位置

额外要求：

- “已训练时长”需要保留，但不应单独霸占一整块大区域
- 它应并入次数信息区，作为右侧次主指标存在

## 6. 历史列表页与详情页

### 6.1 历史列表页定位

列表页只负责快速查找，不负责展示全部详情。

每条记录应压缩为一条可点击记录。

列表项只显示：

- 艺与式
- 总组数与总次数
- 每组次数摘要
- 日期时间

其中信息主次必须明确：

- 每组次数摘要比日期时间更重要
- 日期时间应该缩小、弱化、后退

### 6.2 列表页的后续能力

目前只读并不合理，后续必须支持：

- 编辑
- 删除
- 复制为新训练

首版静态稿中应明确体现这一预期，但不应该在每条记录上同时摆出多个文字按钮。

推荐方式：

- 列表项点击进入详情
- 管理动作收纳到长按或尾部菜单

### 6.3 详情页

详情页继续承接完整信息，但也应预留管理操作：

- 编辑记录
- 删除记录

每组详情的表达要求：

- 不要把“次数 / 时长 / 组间休息 / 结束时间”挤成一整句长文本
- 应拆成清楚的两到三层：
  - 第几组 + 次数
  - 时长 + 组间休息
  - 结束时间
- 组间休息时间与结束时间都需要保留，因为它们属于训练记录的一部分

## 7. 动作标准页

### 7.1 页面定位

动作标准页是训练辅助页，不是整章阅读页。

### 7.2 内容结构

建议固定为：

1. 当前艺与式
2. 原书来源章节
3. 标准说明
4. 动作要点
5. 常见错误
6. 示意图位
7. 开始训练

### 7.3 内容来源

动作标准页未来接入内容时，应以用户提供的 EPUB 为准。

已确认章节映射：

- 俯卧撑：`text00013.html`
- 深蹲：`text00014.html`
- 引体向上：`text00015.html`
- 举腿：`text00016.html`
- 桥：`text00017.html`
- 倒立撑：`text00018.html`

### 7.4 十式正式名称

本轮必须弃用泛化的 `第 1 式` 占位文本。

已从 EPUB 提取并确认正式名称。

例如：

- 俯卧撑第 4 式：半俯卧撑
- 深蹲第 10 式：单腿深蹲
- 桥第 10 式：铁板桥
- 倒立撑第 10 式：单臂倒立撑

## 8. 本轮验收标准

只有同时满足以下条件，才算这轮静态方向过关：

1. 首页已经明显脱离通用卡片 App 风格
2. 首页十式切换不再出现十个数字按钮或十个均质按键
3. 十式正式名称已进入选择层与详情层
4. 训练进行页的实时信息已经图形化，且“当前组次数 / 累计次数 / 已训练时长”处于同一区域
5. 历史列表已把每组次数摘要提升到比日期时间更高的层级
6. 历史详情已清楚拆出每组的时长、组间休息和结束时间
7. 动作标准页已经具备承载 EPUB 内容的结构

## 9. 官方下一步

1. 当前静态原型 `prototype/ui-redesign.html` 已通过本轮 UI 确认
2. 后续 Android Compose 页面必须按该稿逐页落地
3. 如 Android 当前实现与本蓝图或 HTML 稿冲突，以 HTML 稿为准修正
