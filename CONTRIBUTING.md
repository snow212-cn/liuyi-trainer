# 贡献指南

感谢你愿意改进 Liuyi Trainer。

## 基本原则

- 优先在 `develop` 分支开展日常开发
- `main` 只接收准备公开展示或发布的改动
- 每个 PR 只解决一个明确问题，保持改动集中
- 提交前说明改动目的、影响范围和验证方式

## 议题与 PR

1. 先通过 issue 说明问题、需求或设计方向
2. 认领后再提交 PR，避免重复开发
3. PR 描述至少包含：背景、改动、验证、风险

## 代码要求

- 保持 Kotlin 与 Compose 代码可读
- 优先做小步提交，不把重构、功能和格式化混成一次
- 涉及数据结构或页面流程变化时，同步更新对应文档

## 测试要求

- 能跑本地构建时，至少说明是否执行过 `./gradlew assembleDebug`
- 如果本地环境不完整，请写明未执行的验证项和原因
- 影响训练流程、历史记录、导入导出时，补充人工验证步骤

## 内容来源

当前仓库允许保留构建 APK 所需的训练正文、插图和配套素材，但要遵守两条规则：

- 不要删除现有构建关键内容，除非已经确认有替代方案并完成验证
- 新增或替换素材时，必须补充来源说明，避免把来源不明的内容混入主线

## 安全问题

不要用公开 issue 披露会影响用户数据、签名密钥、构建分发链路的细节。此类问题请按 `SECURITY.md` 处理。
